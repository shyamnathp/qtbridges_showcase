Recap: QtBridges in May 2025

  • QtBridges API – part of PySide6.QtQml
  • Exposing Python data model to QML
    • Public Methods → QML Slots
    • Python Properties → QML Properties (+ propertyNameChanged signal)
    • data() → Model data
    • updateQML() → Model updates via beginResetModel()/endResetModel()
    • AutoQmlBridge() → Automatic QML exposure (creates singleton)

from PySide.Qml import AutoQmlBridge, updateQML

class StringModel:
  def __init__(self):
  self._items = ["Apple", "Banana"]

  @property
  def items(self) -> list[str]:
  return self._items

  @items.setter
  def items(self, value: list[str]):
  self._items = value

  def add_string(self, new_str: str):
  self._items.append(new_str)
  updateQML(self)

  def data(self) -> list[str]:
  return self._items

# Register with QML
string_model = StringModel()
AutoQmlBridge(string_model)

What's New in QtBridges for Python (1/2)

Now a standalone project, independent of PySide6.QtQml

  • Key challenges faced:
    • Find a suitable build backend that uses CMake
    • Reuse helper code from Shiboken/PySide
      • DynamicMetaObjectBuilder
      • PySide's Slot/Property system
      • Python/C++ conversion utilities
    • Cross-platform compatibility (Windows, Linux, macOS)
    • Build backend must support Limited-API
1. Why do we write the core bridge code in C++ instead of Python? - To directly access Qt's internal classes and meta-object system, which are only available in C++. - To leverage Shiboken's advanced helper functions for Python/C++ type conversion and memory management. - To use tools like DynamicMetaObjectBuilder for runtime registration of methods, properties, and signals. - This enables seamless integration with QML and high performance, which would not be possible with a pure Python implementation.

Limited API - What & Why?

What is Limited API?

  • Stable subset of Python C API
  • Guarantees ABI compatibility across Python versions
  • Introduced in Python 3.2, stabilized in 3.9+

Benefits

  • Single wheel for Python 3.9-3.12
  • Reduced maintenance burden
    • No need to rebuild for each Python version
    • Fewer binary packages to maintain
  • Future-proof ABI compatibility
  • Smaller binary size
Limited API ensures QtBridges works across Python versions without recompilation. Critical for distribution and maintenance.

Build backend: scikit-build-core

  • Modern Python build backend for C++ extensions
  • Integrates CMake with Python packaging
  • Cross-platform and supports Limited-API
  • Produces ready-to-install wheels for all platforms

What's New in QtBridges for Python? (2/2)

  • Replaces @updateQML decorator
  • Granular model updates with new Python decorators:
    • @insert – beginInsertRows(), endInsertRows()
    • @remove – beginRemoveRows(), endRemoveRows()
    • @move – beginMoveRows(), endMoveRows()
Having just updateQML() meant having parameters to specify in updateQML syntax whether insert, remove or move

@insert Decorator

  • Use @insert to mark methods that insert new items into your model.
  • Automatically triggers granular model updates in QML views.

class StringModel:
  ...

  @insert
  def add_string(self, value: str):
      if value in self._items:
          return False
      self._items.append(value)
      return True

  @insert
  def insert_at(self, index: int, value: str):
      if value in self._items:
          return False
      self._items.insert(index, value)
      return True

@remove Decorator

  • Use @remove to mark methods that remove items from your model.
  • Will enable granular removal updates in QML views.

class StringModel:
    ...

    @remove
    def remove_at(self, index: int) -> bool:
        """Remove item at specific index"""
        if 0 <= index < len(self._items):
            self._items.pop(index)
            return True
        return False

@move Decorator

  • Use @move to mark methods that move items within your model.
  • Will enable granular move updates in QML views.

class StringModel:
  ...

  @move
  def move_item(self, from_index: int, to_index: int):
      if 0 <= from_index < len(self._items)
         and 0 <= to_index < len(self._items):
          item = self._items.pop(from_index)
          self._items.insert(to_index, item)
          return True
      return False

Data Mapping in QtBridges

  • Type mapping: same as PySide6
  • Conversion handled by Shiboken
  • Python containers (list, dict) → Qt types (QVariantList, QVariantMap)
  • Built-in types (int, float, str, bool, None) handled directly
  • Custom/complex types: Shiboken converters

Typing & .pyi Files in Python

Why Typing?

  • Python is dynamically typed—no compile-time type checks
  • Type hints improve code completion, static analysis, and error detection
  • Helps IDEs (PyCharm, VSCode) provide better developer experience

What are .pyi Files?

  • Stub files containing type information for compiled modules
  • Required for tools like mypy and IDEs to understand Python/C/C++ extensions
  • Included in the wheel (see py.typed marker)
  • Enables static type checking for your Python API
- Explain that compiled languages (C++, Rust) have type info at compile time. - Python needs .pyi files for static analysis and IDE support.
